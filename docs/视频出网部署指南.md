# 机器狗视频出网部署指南

## 架构

```
 工地局域网                       公网
┌──────────┐  RTSP   ┌──────────────┐   ffmpeg    ┌──────────────┐
│ 山猫M20  │ ──────→ │   网关 PC     │ ─────────→  │     VPS      │
│          │  :8554  │              │  RTMP推流   │  (公网IP)    │
└──────────┘         │ 双网卡:      │             │              │
                     │  内网连机器狗 │             │ mediamtx     │
                     │  外网连WiFi  │             │ :1935 RTMP   │
                     └──────────────┘             │ :8888 HLS    │
                                                  │ :8889 WebRTC │
                                                  └──────┬───────┘
                                                         │
                                                 远端浏览器观看
```

**三条链路互不干扰：**

| 链路 | 路径 | 谁用 |
|------|------|------|
| 本地控制 | 浏览器 → FastAPI :8000 → UDP → 狗 | 现场操作员 |
| 远程控制 | 云平台 → MQTT → 桥接 → HTTP → 狗 | 远程控制 |
| 视频推流 | 狗 RTSP → ffmpeg → VPS → 浏览器 | 远程观看 |

> ffmpeg 推流由 systemd 管理，与 Python/FastAPI 完全解耦，互不影响。

## 一、准备

### VPS

买一台最便宜的 Linux VPS（Ubuntu 22.04 / Debian 12）：

| 服务商 | 配置 | 月费 |
|--------|------|------|
| 腾讯云轻量 | 1核1G、3Mbps | ~28元 |
| 阿里云轻量 | 1核1G、3Mbps | ~34元 |

> 3Mbps 够 1-2 人同时看。多人并发需升配带宽。

买完后得到 VPS 公网 IP，记作 `<VPS_IP>`。

### 网关 PC 网络确认

网关 PC 已具备双网访问能力，不需要额外硬件：

```bash
# 确认能访问机器狗
ping 10.21.31.103

# 确认能访问外网
curl -4 ifconfig.me
```

## 二、VPS 部署 mediamtx

mediamtx 是开源 RTSP/RTMP/WebRTC 网关，一个二进制文件即可运行，无需 Docker。

### 2.1 安装

```bash
ssh root@<VPS_IP>

mkdir -p /opt/mediamtx && cd /opt/mediamtx
wget https://github.com/bluenviron/mediamtx/releases/download/v1.11.3/mediamtx_v1.11.3_linux_amd64.tar.gz
tar xzf mediamtx_v1.11.3_linux_amd64.tar.gz
```

### 2.2 配置

编辑 `mediamtx.yml`，只改这几个字段：

```yaml
rtmpAddress: :1935
hlsAddress: :8888
webrtcAddress: :8889

paths:
  robot-dog:
    source: rtmpServer
```

### 2.3 防火墙

**VPS 控制台安全组** 和 **系统防火墙** 都要放行：

```bash
ufw allow 1935/tcp   # RTMP 推流
ufw allow 8888/tcp   # HLS 播放
ufw allow 8889/tcp   # WebRTC 播放
ufw allow 22/tcp     # SSH
ufw enable
```

### 2.4 设为系统服务

```bash
cat > /etc/systemd/system/mediamtx.service << 'EOF'
[Unit]
Description=mediamtx
After=network.target

[Service]
ExecStart=/opt/mediamtx/mediamtx /opt/mediamtx/mediamtx.yml
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target
EOF

systemctl daemon-reload
systemctl enable --now mediamtx
systemctl status mediamtx   # 确认 active
```

## 三、网关 PC 推流

### 3.1 安装 ffmpeg

```bash
# macOS
brew install ffmpeg

# Ubuntu
sudo apt install ffmpeg -y
```

### 3.2 验证本地能拉到视频

```bash
ffplay rtsp://10.21.31.103:8554/video1
# 看到画面 → Ctrl+C 退出，继续下一步
```

### 3.3 手动推流测试

```bash
ffmpeg -rtsp_transport tcp \
  -i rtsp://10.21.31.103:8554/video1 \
  -c:v libx264 -b:v 1500k -maxrate 2000k -bufsize 3000k \
  -preset ultrafast -tune zerolatency \
  -c:a aac -b:a 64k -ar 44100 -ac 1 \
  -f flv \
  rtmp://<VPS_IP>:1935/robot-dog
```

终端持续输出 `speed=1x` 且无报错 → 推流成功。

### 3.4 设为系统服务（macOS 用 launchd）

**Ubuntu（systemd）：**

```bash
cat > /etc/systemd/system/dog-video-push.service << 'EOF'
[Unit]
Description=机器狗视频推流
After=network.target

[Service]
ExecStart=/usr/bin/ffmpeg \
  -rtsp_transport tcp \
  -i rtsp://10.21.31.103:8554/video1 \
  -c:v libx264 -b:v 1500k -maxrate 2000k -bufsize 3000k \
  -preset ultrafast -tune zerolatency \
  -c:a aac -b:a 64k -ar 44100 -ac 1 \
  -f flv \
  rtmp://<VPS_IP>:1935/robot-dog
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target
EOF

systemctl daemon-reload
systemctl enable --now dog-video-push
```

**macOS（launchd）：**

```bash
cat > ~/Library/LaunchAgents/com.dog.video-push.plist << 'EOF'

    <key>Label</key>

    <array>
        <string>/opt/homebrew/bin/ffmpeg</string>
        <string>-rtsp_transport</string><string>tcp</string>
        <string>-i</string><string>rtsp://10.21.31.103:8554/video1</string>
        <string>-c:v</string><string>libx264</string>
        <string>-b:v</string><string>1500k</string>
        <string>-maxrate</string><string>2000k</string>
        <string>-bufsize</string><string>3000k</string>
        <string>-preset</string><string>ultrafast</string>
```

## 四、远端播放页面

VPS 上装 nginx 托管播放页面：

```bash
ssh root@<VPS_IP>
apt install nginx -y
```

创建 `/var/www/html/dog-video.html`：

```html
<!DOCTYPE html>
<html lang="zh">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>机器狗实时视频</title>
  <style>
    body { margin: 0; background: #000; display: flex; justify-content: center;
           align-items: center; min-height: 100vh; }
    video { max-width: 100%; max-height: 100vh; }
  </style>
</head>
<body>
  <video id="video" controls autoplay muted></video>
  <script src="https://cdn.jsdelivr.net/npm/hls.js@latest"></script>
  <script>
  const video = document.getElementById("video");
  const VPS_IP = window.location.hostname;
  const hlsUrl = `http://${VPS_IP}:8888/robot-dog/`;
  if (Hls.isSupported()) {
    const hls = new Hls({ liveSyncDurationCount: 1 });
    hls.loadSource(hlsUrl);
    hls.attachMedia(video);
  } else {
    video.src = hlsUrl;
  }
  </script>
</body>
</html>
```

远端访问：`http://<VPS_IP>/dog-video.html`

> 手机切 4G/5G 打开链接即可验证，延迟约 3-5 秒。

## 五、联调检查清单

| # | 检查项 | 命令/操作 | 预期 |
|---|--------|----------|------|
| 1 | 机器狗 RTSP | `ffplay rtsp://10.21.31.103:8554/video1` | 有画面 |
| 2 | VPS mediamtx | `ssh root@<VPS_IP> systemctl status mediamtx` | active |
| 3 | VPS 端口可达 | `nc -zv <VPS_IP> 1935` | succeeded |
| 4 | 推流状态 | `journalctl -u dog-video-push -f` | 日志有 `speed=1x` |
| 5 | 远端播放 | 手机 4G 打开 `http://<VPS_IP>/dog-video.html` | 有画面 |

### 排障

```bash
# 推流失败 — VPS 端口不通
nc -zv <VPS_IP> 1935

# 推流成功但远端黑屏 — 看 mediamtx 日志
ssh root@<VPS_IP> journalctl -u mediamtx -f

# 视频卡顿 — 降低码率，把 -b:v 改成 1000k
```
